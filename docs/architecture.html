Axios Auto-Refresh Flow (axiosInstance.js)
When any API call returns 401 (token expired), the interceptor automatically refreshes and retries:
1

API request fires → 401 response

Token has expired. Interceptor catches the error.

2

Check if already refreshing

If yes, queue this request in refreshQueue[] (prevents concurrent refresh race condition).

3

POST /api/auth/refresh

Send refreshToken from localStorage. Receive new accessToken.

4

Store new token + drain queue

Save to localStorage, update Auth header, replay all queued requests with new token.

5

Refresh fails → logout

Clear tokens, redirect to /login.

// axiosInstance.js key logic let isRefreshing = false; let refreshQueue = []; const drainQueue = (token) => { refreshQueue.forEach(cb => cb(token)); refreshQueue = []; }; // 401 interceptor if (error.response?.status === 401) { if (isRefreshing) { // Queue this request return new Promise(resolve => { refreshQueue.push(token => { original.headers.Authorization = `Bearer ${token}`; resolve(axiosInstance(original)); }); }); } isRefreshing = true; try { const res = await authApi.refresh( { refreshToken } ); const newToken = res.data.data.accessToken; localStorage.setItem('token', newToken); drainQueue(newToken); return axiosInstance(original); } catch { logout(); // clear + redirect } finally { isRefreshing = false; } }

🔒 Security Architecture

Multi-layer security: JWT stateless auth, BCrypt password hashing, TOTP 2FA, CORS policy, and session management.

Authentication Flow
1

POST /api/auth/login

User sends email + password. Spring Security validates via UserDetailsServiceImpl.loadUserByUsername().

2

BCrypt.matches()

PasswordEncoder compares submitted password with stored hash.

3

2FA check (if enabled)

TOTP code validated against stored secret using RFC 6238.

4

Token generation

JwtUtils.generateAccessToken() + generateRefreshToken(). Session stored in MongoDB.

5

Response

Returns { accessToken, refreshToken, user }. Frontend stores tokens in localStorage.

JWT Request Validation
1

JwtAuthFilter.doFilterInternal()

Extracts Bearer token from Authorization header on every request.

2

JwtUtils.validateToken()

Verifies HMAC-SHA256 signature, checks expiry, extracts subject (email).

3

UserDetailsServiceImpl.loadUserByUsername(email)

Fetches user from MongoDB to confirm account is still active.

4

SecurityContextHolder.setAuthentication()

Controller calls authentication.getName() → gets email → service resolves to userId.

🔄 Request Flow Diagrams

Step-by-step flows for the most important operations in the application.

Create + Auto-Save Note Flow
FE

User types in NoteEditorPage

Title/content change → scheduleAutoSave() sets 2s debounce timer.

FE

Timer fires → saveNote()

If new note: POST /api/notes. If existing: PUT /api/notes/{id}.

BE

NoteController.createNote()

Calls noteService.createNote(request, email). resolveUserId(email) → userId.

BE

NoteService builds Note model

Sets wordCount (strip HTML tags + split), version=1, isDeleted=false.

DB

noteRepository.save(note)

MongoDB inserts document. @CreatedDate / @LastModifiedDate auto-set.

FE

URL updates to /notes/{newId}

navigate() with replace:true. Saved indicator turns green ✓.

Collaboration Invite + Access Flow
FE

Owner opens CollaborateModal

Enters user@email.com + READ/WRITE permission, clicks Invite.

BE

POST /api/collaborate/{noteId}/invite

CollaborateService.invite() → finds invitee by email → checks not self-invite.

DB

Upserts collaborator in note.collaborators[]

If already collaborator → updates permission. Otherwise appends new Collaborator.

FE

Invited user logs in → visits /shared-with-me

GET /api/notes/shared-with-me → query: { 'collaborators.userId': userId }

BE

Open note → findNoteWithAccess()

Checks owner OR collaborator with sufficient permission (READ/WRITE).

FE

Editor editable based on permission

WRITE: full editing. READ: locked editor + blue "read-only" banner.

Share Link Flow
FE

Owner clicks Share button

POST /api/share/{noteId} with body { permission: "VIEW" }

BE

ShareService.createShareLink()

Generates UUID token. Stores SharedLink document. Returns http://localhost:3000/share/{token}

FE

Modal shows copyable URL

Owner copies and sends to anyone. No login required to view.

FE

Recipient opens link in browser

Route /share/:token → SharedNotePage → GET /api/share/public/{token}

BE

ShareService.getSharedNote(token)

Finds SharedLink → validates isActive → returns NoteResponse. Endpoint is unauthenticated.

Search Flow
FE

User types in SearchPage

useDebounce(query, 400ms) prevents firing on every keystroke.

BE

GET /api/notes?search={query} or /search?q={q}

NoteService.searchNotes() → NoteRepository.searchByTitleOrContent(userId, query)

DB

MongoDB regex query

{ $or: [{ title: { $regex: query, $options: 'i' } }, { content: { $regex: query, $options: 'i' } }] }

FE

Results with highlights

highlightText() wraps matches in <mark> tags. Dark/light aware colors.

✨ Feature Reference

Complete list of all implemented features with their technical implementation details.

✏️
Rich Text Editor
TipTap v2 with extensions: Bold, Italic, Strike, Code, Heading (H1-H3), BulletList, OrderedList, Blockquote, Link, Horizontal Rule. Floating toolbar. Content stored as HTML.
💾
Auto-Save
2-second debounce on title and content changes. Saves indicator: Unsaved → Saving… → ✓ Saved. New note creates on first save and updates URL with replace:true.
📁
Folder Organization
Create, rename, delete folders from sidebar. Notes filtered by folder. Nested folder support via parentId. Note count displayed per folder.
🔍
Full-Text Search
MongoDB regex search across title and content. Debounced input (400ms). Match highlighting with <mark> tags. Separate /search page with result cards.
📌
Pin & Favourite
Toggle pin/favourite on any note. Pinned notes float to top. Dedicated /favourites page. Dashboard filter buttons for pinned/favourite notes.
🗑️
Trash & Restore
Soft delete moves notes to trash (isDeleted=true, deletedAt recorded). Restore clears isDeleted. Permanent delete removes from MongoDB. Empty Trash clears all at once.
🔒
Note Lock
Set a password to lock a note. Password stored as BCrypt hash. Locked note shows read-only editor (TipTap editable=false). Unlock requires matching password.
🔗
Public Share Links
Generate UUID share token. Public URL: /share/{token}. No auth required. SharedNotePage renders note in read-only TipTap. Link can be revoked (isActive=false).
👥
Real Collaboration
Invite by email with READ or WRITE permission. Collaborators appear in /shared-with-me. READ: read-only view with banner. WRITE: full editing. Owner can remove collaborators.
🤖
AI Assistant
Side panel with 4 features: Summarize content, Grammar correction, Rewrite with custom instruction, Suggest tags. Endpoints stubbed — ready for OpenAI/Anthropic integration.
📤
Export Notes
Export as Markdown (.md) or plain text (.txt). Server strips HTML tags for text export. TipTap HTML converted to Markdown format. Returned as file download (Content-Disposition header).
🌓
Dark / Light Theme
Tailwind darkMode: 'class'. Toggle adds/removes .dark on <html>. ThemeContext persists to localStorage. Custom CSS overrides in global.css for components using base dark utilities.
🔐
Two-Factor Auth (2FA)
TOTP-based (RFC 6238). Compatible with Google Authenticator. Setup: generate secret + QR code. Verification: 6-digit code on login. Stored as Base32 secret in user document.
📧
Password Reset
Forgot password sends 6-digit OTP via SMTP email. OTP stored in MongoDB with 15-minute expiry. Reset form: email + OTP + new password. BCrypt encodes new password.
🖥️
Session Management
Active sessions stored in MongoDB with device info, IP, last activity. View all sessions in Settings. Revoke individual sessions or log out all devices. JWT refresh validates against session record.
📋
Audit Logs
All important actions logged: note CRUD, login/logout, password changes, collaborator changes. Viewable in Settings → Security tab. Includes action, resource, IP address, timestamp.
📄
Swagger / OpenAPI
Springdoc OpenAPI 2.3.0. All endpoints documented with @Operation and @Tag annotations. Accessible at http://localhost:8081/swagger-ui.html. Supports JWT Bearer token testing.
🔢
Note Metadata
Auto word count (strips HTML, splits on whitespace). View count incremented on each GET. Version number increments on each update. Created/updated timestamps via Spring Data auditing.
